# 微内核操作系统学习与开发规约 (v3.0)

## 1. 学习目标
- 掌握从 CPU 上电到内核运行的完整生命周期。
- 深入理解 x86_64 保护模式、分页机制与中断处理。
- 实践核心数据结构（如红黑树）在内存管理中的应用。
- 实现微内核特有的 IPC 机制。

## 2. 阶段化开发路线 (Agent 需按阶段引导)
- **第一阶段：引导与裸机打印**。实现 Multiboot2 引导，通过串口/VGA 打印。
- **第二阶段：GDT 与 中断 (IDT)**。设置全局描述符表，接管 CPU 中断。
- **第三阶段：物理与虚拟内存管理**。实现位图分配器与页表映射。
- **第四阶段：红黑树 VMA 管理**。用红黑树管理进程地址空间（关键学习点）。
- **第五阶段：进程切换与多任务**。保存上下文，实现时钟中断驱动的调度。
- **第六阶段：微内核 IPC**。实现同步消息传递。

## 3. 技术约束 (针对学习者)
- **底层可见性**: 尽量使用内联汇编或显式内存操作，避免过度的抽象封装。
- **注释驱动**: 代码必须包含对 x86 架构特性的详细解释（如：CR3 寄存器、页表项位定义）。
- **无依赖**: 不使用任何外部库，所有逻辑自足。

### 3.1 学习级注释规范（后续默认要求）
- 对涉及 CPU/体系结构的代码（如：GDT/IDT、分页、寄存器约定、启动协议），必须写“学习级”注释：
    - 写清楚“为什么需要这一步”（目的/后果/不做会怎样）。
    - 写清楚“CPU 在做什么”（关键寄存器、隐式缓存、指令副作用）。
    - 写清楚“数据结构每一位/每一字段的含义”（例如 access/granularity 位）。
    - 写清楚“验证方法”（怎样在 QEMU 串口看到正确行为）。

## 4. 调试与验证
- 必须支持通过串口 (0x3F8) 输出调试信息，以便在 QEMU 终端查看。

## 5. 更新方案
- 在每次agent更新以后，在docs/changelog.md里面更新做了什么
