# Book 子项目：《自己动手写操作系统》维护规约

当用户说"更新书"、"写书"、"补充章节"等关键词时，按以下流程操作。

## 1. 信息采集（必须按顺序）

1. **读 changelog**：`docs/changelog.md`，确定自上次更新以来有哪些新的 Stage/功能完成
2. **读 roadmap**：`docs/roadmap.md`，了解当前进度和 Stage 编号
3. **读已有章节**：`book/chapters/` 下已有的 `.tex` 文件，确定哪些章节已写、写到什么程度
4. **读对应源码**：根据需要补充的章节，读取对应的 `.h`、`.c`、`.asm` 文件中的注释和实现
5. **读 shell 命令文档**：`docs/memory-commands.md`，获取验证示例

## 2. 章节内容标准

每个章节（对应一个 Stage）必须包含以下结构：

```latex
\chapter{标题}
\section{本章目标}          % 要点列表：这一步做什么
\section{背景知识}          % 硬件/CPU 原理，带图/表/bitfield
\section{设计决策}          % 为什么选这种方案（用 designbox/whybox 环境）
\section{数据结构}          % 关键 struct/typedef 的逐字段解释
\section{实现详解}          % 核心函数的逐步讲解，含代码片段
\section{CPU 状态变化}      % 用 cpustate 环境说明（如果涉及特权/分页/中断）
\section{验证}              % shell 命令输出示例或 QEMU 观察方法
\section{本章小结}          % 一段话总结 + 引出下一章
```

不是每个 section 都必须有（比如不涉及 CPU 状态变化就跳过），但**目标、实现详解、验证**是必须的。

## 3. 代码引用规则

- 关键代码用 `lstlisting` 环境内嵌（不要引用外部文件路径）
- C 代码用 `style=cstyle`，汇编用 `style=nasmstyle`，shell 用 `style=shellstyle`
- 代码注释中的 `[WHY]`、`[CPU STATE]`、`[BITFIELDS]` 标签内容必须搬入正文讲解
- 不要复制整个文件——只引用核心片段，用自然语言串联

## 4. 自定义环境使用

| 环境 | 用途 | 示例场景 |
|------|------|----------|
| `designbox` | 设计决策 | "为什么用红黑树而不是链表" |
| `whybox` | 原理解释 | "为什么内核要加载到 1MiB" |
| `warnbox` | 易错警告 | "拆除 identity mapping 前必须转换所有指针" |
| `cpustate` | CPU 状态变化 | "lgdt 后 CS 缓存更新" |
| `bitfield` + `\bit` | 寄存器/PTE 位域 | PDE/PTE 各 bit 含义 |
| `thinkbox` | 思考题（穿插正文） | "如果 identity mapping 少映射了一页会怎样？" |
| `exercise` | 动手练习（章末） | "修改波特率为 115200 并验证输出" |

### 思考题与练习规则

- **思考题 (`thinkbox`)**：穿插在正文讲解中，出现在关键设计决策或容易混淆的概念之后。引导读者停下来思考"为什么"。每章 2-4 个。
- **动手练习 (`exercise`)**：放在章末"本章小结"之前。包含具体操作步骤，读者可以在 QEMU 中验证。每章 2-5 个。
- 练习难度递增：简单（修改参数观察变化）→ 中等（分析输出）→ 挑战（独立实现小功能）

## 5. 进度跟踪

- 章节中有 `% TODO:` 注释的表示框架已搭但内容未填
- 填充内容后删除对应 `% TODO:` 注释
- 新 Stage 完成后，先在 `main.tex` 添加 `\include{}`，再创建章节文件

## 6. 构建环境与验证

### 首次构建前：安装 LaTeX 依赖

在构建 PDF 之前，必须确认 XeLaTeX 和中文支持已安装。如果用户要求构建或用户环境中 `xelatex` 不存在，先执行安装：

```bash
# 检查是否已安装
which xelatex || echo "需要安装"

# Ubuntu / WSL2 安装命令（一次性）
sudo apt update
sudo apt install -y \
  texlive-xetex \
  texlive-lang-chinese \
  texlive-latex-extra \
  texlive-latex-recommended \
  texlive-fonts-recommended \
  texlive-fonts-extra \
  texlive-science \
  latexmk
```

各包用途：
- `texlive-xetex` — XeLaTeX 引擎（支持 UTF-8 + 系统字体）
- `texlive-lang-chinese` — ctex 宏包（中文排版核心）
- `texlive-latex-extra` — tcolorbox、listings、fancyhdr 等扩展包
- `texlive-latex-recommended` — booktabs、longtable、geometry 等常用包
- `texlive-fonts-recommended` — 推荐字体集
- `texlive-fonts-extra` — 额外字体（确保中文字体可用）
- `texlive-science` — amsmath 等数学包
- `latexmk` — 自动化构建工具（`make watch` 依赖）

### 构建命令

每次修改后运行：
```bash
cd book && make once    # 快速编译一遍（预览）
cd book && make         # 完整编译（两遍，生成目录）
cd book && make clean   # 清理构建产物
cd book && make watch   # 持续监听自动重编译（需要 latexmk）
```

确认无 LaTeX 编译错误后再提交。

## 7. 文件命名约定

- 章节文件：`chapters/chNN-短描述.tex`（NN 从 00 开始）
- 图片文件：`figures/chNN-描述.png`（或 .pdf/.svg）
- 编号和 main.tex 中的 `\include` 顺序一致
