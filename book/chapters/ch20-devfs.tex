\section{本章目标}

\begin{itemize}
  \item 理解 Unix 设备模型（主设备号 / 次设备号）
  \item 设计可插拔 \texttt{chardev\_ops\_t} 接口 + cdev 注册框架
  \item 实现基础字符设备：\texttt{/dev/null}、\texttt{/dev/zero}、\texttt{/dev/serial}、\texttt{/dev/console}
  \item 通过 VFS 统一访问设备文件
  \item 里程碑 E 集成测试
\end{itemize}

\section{本章小结}
% ============================================================================

字符设备通过可插拔 \texttt{chardev\_ops\_t} 接口纳入 VFS ``一切皆文件''体系。
至此里程碑 E（文件系统）全部完成：VFS 抽象层 + ramfs/initrd + ext2 磁盘 + 字符设备 四层递进。
下一部分进入 POSIX 兼容层——信号、管道、mmap、libc 和用户态 Shell。
