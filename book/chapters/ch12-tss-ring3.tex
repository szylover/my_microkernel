\end{itemize}

\section{跳转到用户态：\texttt{iret}}
\label{sec:tss-iret}
% ============================================================================

% TODO:
% - 目标：从 Ring 0 跳到 Ring 3 执行一段测试代码
% - iret 帧构造（栈上从低到高）：
%     push SS3  (0x23 = user data)
%     push ESP3 (用户态栈顶)
%     push EFLAGS (IF=1 保持中断)
%     push CS3  (0x1B = user code)
%     push EIP3 (用户态入口地址)
%     iret
% - [CPU STATE] iret 执行后的完整状态变化：
%     CS.RPL: 0 → 3, CPL: 0 → 3
%     SS: 0x10 → 0x23, ESP: kernel_stack → user_stack
%     EIP: 指向用户态函数
%     页表 PTE_USER 位控制哪些页用户态可访问
% - 用户态栈分配：kmalloc 或 pmm_alloc 一页 → 映射到用户地址空间 → 设 PTE_USER
% - 汇编辅助函数 jump_to_ring3(eip, esp)

\begin{cpustate}
\texttt{iret} 执行后：
\begin{itemize}
  \item CPL 从 0 变为 3（用户态）
  \item CS 指向用户代码段 (0x1B)，SS 指向用户数据段 (0x23)
  \item ESP 切换到用户态栈
  \item 内核页表中 PTE\_USER 位控制哪些页用户态可访问
  \item 后续中断发生时，CPU 自动从 TSS 读取 SS0:ESP0 切回内核栈
\end{itemize}
\end{cpustate}

\section{本章小结}
% ============================================================================

TSS + GDT 扩展 + \texttt{iret} 让我们具备了 Ring 0 ↔ Ring 3 双向切换的能力。
但目前用户态代码无法请求内核服务——下一章实现系统调用机制，
在 Ring 3 和 Ring 0 之间建立受控通信通道。
