\section{本章目标}

\begin{itemize}
  \item 将内核映射到虚拟地址 \hex{C0000000}+（高半区）
  \item 拆除低地址 Identity Mapping，为用户态腾出 0--3GiB 空间
  \item 所有指针从物理地址切换到虚拟地址
\end{itemize}

\section{背景知识：High-Half Kernel 架构}

% TODO:
% - 地址空间划分：0x00000000-0xBFFFFFFF (用户) / 0xC0000000-0xFFFFFFFF (内核)
% - 直接映射区: virt = phys + 0xC0000000
% - PHYS_TO_VIRT / VIRT_TO_PHYS 宏

\begin{whybox}
为什么要 High-Half？让每个用户进程都有完整的 0--3GiB 虚拟地址空间，
同时内核在所有进程的页表中都占据相同的高地址区域。
切换进程时只需换低 3GiB 的映射，内核部分不变。
\end{whybox}
