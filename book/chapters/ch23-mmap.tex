\section{本章目标}

\begin{itemize}
  \item 理解 mmap 的用途与语义（匿名映射 / 文件映射）
  \item 实现 mmap 与 VMA 子系统的集成（复用里程碑 C 的可插拔 VMA 后端）
  \item 实现匿名映射后端（\texttt{MAP\_ANONYMOUS | MAP\_PRIVATE}）
  \item 实现文件映射后端（进阶，\texttt{MAP\_PRIVATE} 文件映射）
  \item 按需分页（demand paging）与 Page Fault 处理
  \item 通过 \texttt{KCONFIG\_MMAP\_FILE} 开关控制文件映射
\end{itemize}

\section{本章小结}
% ============================================================================

mmap 将虚拟地址映射与 VMA 子系统深度集成，实现了 demand paging 的按需分配。
匿名映射为用户态 \texttt{malloc} 的大块分配提供基础，文件映射为动态库和高效 I/O 铺路。
下一章实现用户态 libc，让用户程序能用标准 C 函数编程。
