\section{架构设计}

Shell 分五层，每层职责清晰：

\begin{lstlisting}[style=shellstyle]
输入层 (console.c)     keyboard IRQ1 + serial 轮询 → console_getc()
        |
行编辑层 (shell.c)     回显 / Backspace / Enter → line[128]
        |
命令解析 (shell.c)     空格分词 → argv[0]=cmd, argv[1..]=args
        |
命令分发 (shell.c)     遍历 g_cmds[] → 匹配 name → 调用 fn()
        |
命令实现 (cmds/*.c)    每个命令一个 .c 文件，导出 const cmd_t
\end{lstlisting}

\begin{thinkbox}
为什么要抽象 Console 层，而不让 Shell 直接调用 \texttt{keyboard\_getc()}？
思考：如果以后要支持串口输入（远程调试）或网络终端，需要改动几处代码？
\end{thinkbox}

\section{Console 输入层}

\begin{lstlisting}[style=cstyle]
char console_getc(void) {
    for (;;) {
        // 优先查键盘 IRQ 缓冲区（无等待）
        char c;
        if (keyboard_try_getc(&c)) return c;
        // 其次查串口（轮询）
        if (serial_try_getc(&c))   return c;
        __asm__ volatile("hlt");  // 等下一个中断
    }
}
\end{lstlisting}

\section{命令接口：\texttt{cmd.h}}

\begin{lstlisting}[style=cstyle]
typedef int (*cmd_fn_t)(int argc, char** argv);

typedef struct {
    const char* name;   // 命令名
    const char* help;   // 帮助文本
    cmd_fn_t fn;        // 处理函数
} cmd_t;
\end{lstlisting}

每个命令做成独立编译单元，导出 \texttt{const cmd\_t cmd\_xxx}：

\begin{lstlisting}[style=cstyle]
// cmd_cls.c
static int cmd_cls_fn(int argc, char** argv) {
    printk("\x1b[2J\x1b[H");  // ANSI 清屏
    return 0;
}
const cmd_t cmd_cls = { .name = "cls", .help = "Clear screen", .fn = cmd_cls_fn };
\end{lstlisting}

Shell 维护注册表并按 name 分发：

\begin{lstlisting}[style=cstyle]
static const cmd_t* g_cmds[] = { &cmd_cls, &cmd_shutdown, &cmd_cmds, ... };
\end{lstlisting}

\begin{designbox}
为什么用静态数组而非链表？

内核态命令数量有限（十几个），静态数组避免了 kmalloc 依赖（此时堆还没初始化）。
添加新命令只需：写 .c → extern → 加入数组。Makefile 自动发现 \texttt{cmds/*.c}。
\end{designbox}

\section{思考与练习}

\begin{exercise}
添加一个 \texttt{echo} 命令，将 \texttt{argv[1..]} 的所有参数用空格连接后打印。
例如 \texttt{echo hello world} 输出 \texttt{hello world}。
\end{exercise}

\begin{exercise}
修改 Shell，支持上方向键回显上一条命令（命令历史）。
提示：维护一个历史缓冲区 \texttt{history[8][128]}，用环形索引。
键盘的上箭头 scancode 是一个多字节序列（\hex{E0} \hex{48}）。
\end{exercise}

\begin{exercise}
当用户输入不存在的命令时，Shell 当前只是静默忽略。修改为打印
\texttt{unknown command: xxx}。思考：在什么阶段检查？dispatch 函数还是 tokenize 之后？
\end{exercise}
