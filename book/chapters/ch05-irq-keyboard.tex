\section{背景知识：8259 PIC}

x86 PC 有两片级联的 8259 PIC，管理 16 条硬件中断线（IRQ0--15）：

\begin{longtable}{cll}
  \toprule
  \textbf{IRQ} & \textbf{设备} & \textbf{PIC} \\
  \midrule
  0  & PIT（定时器）   & 主 PIC \\
  1  & PS/2 键盘       & 主 PIC \\
  2  & 级联从 PIC      & 主 PIC \\
  8  & CMOS/RTC        & 从 PIC \\
  14 & Primary ATA     & 从 PIC \\
  \bottomrule
\end{longtable}

\subsection{为什么要 Remap？}

PIC 上电后默认将 IRQ0--7 映射到中断向量 8--15，与 CPU 异常重叠！
必须 remap 到 0x20--0x2F（32--47）来避免冲突。

\begin{thinkbox}
如果不做 remap，IRQ1（键盘）会映射到向量 9。向量 9 原本是什么异常？
这会导致什么后果？
\end{thinkbox}

\subsection{ICW 初始化序列}

remap 通过向 PIC 发送 4 个 ICW（Initialization Command Words）完成：

\begin{lstlisting}[style=cstyle]
void pic_init(void) {
    // ICW1: 开始初始化序列
    outb(0x20, 0x11);  outb(0xA0, 0x11);
    // ICW2: 基向量（主 PIC → 0x20，从 PIC → 0x28）
    outb(0x21, 0x20);  outb(0xA1, 0x28);
    // ICW3: 主从级联关系
    outb(0x21, 0x04);  outb(0xA1, 0x02);
    // ICW4: 8086 模式
    outb(0x21, 0x01);  outb(0xA1, 0x01);
    // OCW1: 屏蔽所有 IRQ（后续按需开启）
    outb(0x21, 0xFF);  outb(0xA1, 0xFF);
}
\end{lstlisting}

\subsection{EOI（End of Interrupt）}

每次 IRQ handler 完成后必须向 PIC 发送 EOI，否则 PIC 不会再发送该优先级及以下的中断：

\begin{lstlisting}[style=cstyle]
void pic_send_eoi(uint8_t irq) {
    if (irq >= 8) outb(0xA0, 0x20);  // 从 PIC
    outb(0x20, 0x20);                  // 主 PIC
}
\end{lstlisting}

\begin{warnbox}
忘记发 EOI 是最常见的硬件中断 bug。表现为：第一次按键有反应，之后键盘完全无响应。
\end{warnbox}

\section{IRQ 分发框架：\texttt{irq.c}}

\begin{lstlisting}[style=cstyle]
static irq_handler_t g_irq_handlers[16] = {0};

void irq_install_handler(uint8_t irq, irq_handler_t handler) {
    g_irq_handlers[irq] = handler;
}

void irq_handler_c(regs_t* regs) {
    uint8_t irq = regs->int_no - 0x20;
    if (irq < 16 && g_irq_handlers[irq]) {
        g_irq_handlers[irq](regs);
    }
    pic_send_eoi(irq);
}
\end{lstlisting}

\begin{designbox}
为什么用函数指针数组而非 switch/case？

每个驱动只需调用 \texttt{irq\_install\_handler(1, keyboard\_irq\_handler)} 即可注册。
添加新设备不需要修改 irq.c，符合开放封闭原则。
\end{designbox}

\subsection{Scancode → ASCII}

键盘按下时发送 scancode（Make code），释放时发送 scancode + 0x80（Break code）。
我们维护一张 scancode → ASCII 的查找表（Scancode Set 1）。

\end{thinkbox}

\subsection{环形缓冲区}

\begin{lstlisting}[style=cstyle]
#define KB_BUF_SIZE 256
static char kb_buf[KB_BUF_SIZE];
static volatile uint32_t kb_head = 0;
static volatile uint32_t kb_tail = 0;
\end{lstlisting}

IRQ handler（中断上下文）向 \texttt{kb\_head} 写入字符，
主循环从 \texttt{kb\_tail} 读取。

\begin{designbox}
为什么用环形缓冲区？

中断处理必须尽快返回（否则阻塞其他中断）。环形缓冲区让 IRQ handler 只做
"存字符"，消费由 shell 主循环完成——经典的生产者-消费者模式。
\end{designbox}
