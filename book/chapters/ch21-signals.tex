\section{本章目标}

\begin{itemize}
  \item 理解信号的本质：进程间异步通知机制
  \item 掌握信号生命周期：产生 → 挂起（pending）→ 投递（delivery）→ 处理
  \item 实现 per-process 信号表（31 个标准信号 + pending/blocked 位图）
  \item 实现信号投递机制 + 内核态→用户态信号跳板（signal trampoline）
  \item 实现 \texttt{sigaction} 系统调用（用户自定义信号处理函数）
  \item 验证 \texttt{Ctrl+C} → \texttt{SIGINT} → 终止前台进程
\end{itemize}

\section{本章小结}
% ============================================================================

信号机制为进程提供了异步通知能力。通过 pending/blocked 位图 + signal trampoline，
内核能安全地在用户态执行信号处理函数并恢复原始上下文。
下一章实现管道和 I/O 重定向，构建进程间通信的基础设施。
